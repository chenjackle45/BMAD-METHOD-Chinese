# Story Draft Checklist

Scrum Master 應使用本檢查表，驗證每個 story 是否包含足夠的背景資訊，讓開發代理人能順利實作，並假設開發代理人具備合理的推理能力。

## 1. 目標與背景清晰度

- [ ] 明確說明 story 目標／目的
- [ ] 關聯至 epic 目標清楚
- [ ] 說明 story 在整體系統流程中的定位
- [ ] 識別對前一個 story 的依賴（如適用）
- [ ] 商業背景與價值明確

## 2. 技術實作指引

- [ ] 指定需建立／修改的關鍵檔案（不必窮舉）
- [ ] 說明本 story 需用到的技術
- [ ] 闡述關鍵 API 或介面
- [ ] 參照必要的資料模型或結構
- [ ] 列出所需環境變數（如適用）
- [ ] 註明對標準程式模式的例外

## 3. 參考資料有效性

- [ ] 外部文件參考需指向具體相關章節
- [ ] 摘要前一個 story 的關鍵資訊（不可僅引用）
- [ ] 說明參考資料的相關性
- [ ] 參考格式一致（如 `docs/filename.md#section`）

## 4. 自足性評估

- [ ] 包含所有核心資訊（不可過度依賴外部文件）
- [ ] 隱含假設需明確說明
- [ ] 解釋領域專有名詞或概念
- [ ] 處理邊界情境或錯誤狀況

## 5. 測試指引

- [ ] 說明必要的測試方式
- [ ] 指定關鍵測試情境
- [ ] 定義成功標準
- [ ] 註明特殊測試考量（如適用）

## 驗證結果

| 類別                             | 狀態            | 問題 |
| -------------------------------- | --------------- | ---- |
| 1. 目標與背景清晰度              | PASS/FAIL/PARTIAL |      |
| 2. 技術實作指引                  | PASS/FAIL/PARTIAL |      |
| 3. 參考資料有效性                | PASS/FAIL/PARTIAL |      |
| 4. 自足性評估                    | PASS/FAIL/PARTIAL |      |
| 5. 測試指引                      | PASS/FAIL/PARTIAL |      |

**最終評估：**

- READY：story 已提供足夠實作背景
- NEEDS REVISION：story 需修正（見問題）
- BLOCKED：需外部資訊（請註明所需內容）
