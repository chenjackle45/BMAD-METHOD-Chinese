# 專案簡報：BMad DiCaster

## 簡介 / 問題陳述

本專案旨在開發一個網路應用程式，每日提供 Hacker News (HN) 熱門文章的簡明摘要，並以電子報形式傳送，同時可透過網頁介面存取。

我們試圖解決的問題是，忙碌的專業人士和愛好者希望隨時掌握 HN 的最新動態，卻沒有時間篩選大量的文章和討論，因而面臨資訊過載的困境。此應用程式將透過整理熱門 story、提供 AI 生成的文章和留言摘要，以及可選的 AI 生成 podcast 版本，來簡化 HN 內容的閱覽流程。

解決方案將包含：

* 每日擷取 HN 熱門文章和留言。
* 抓取連結文章。
* AI 生成文章和留言摘要。
* 產生簡明日報。
* 可選的 AI 生成 podcast。
* 透過電子郵件傳送電子報。
* 一個用於存取當期和過往電子報及 podcast 的網頁介面。

## 願景與目標

* **願景：** 打造一個現代網路應用程式的功能展示，利用 AI 進行內容摘要和生成，既作為個人學習經驗，也作為一個實用工具，協助尋求 Hacker News 內容簡明摘要的個人，並為有興趣建構類似應用程式的其他人提供範例。
* **主要目標：**

    1.  **端到端自動化：** 成功自動化每日工作流程，從擷取 HN 文章到寄送電子報郵件，由排程程序觸發，無需任何手動介入。
    2.  **本地/API 觸發：** 在 MVP 階段，實現手動觸發每日工作流程的功能，可在本地透過命令列介面 (CLI) 或透過安全的 API 端點進行。
    3.  **API 安全性：** 確保用於觸發每日工作流程的 API 端點包含基本的安全措施，以防止未經授權的存取。
* **成功指標（初步構想）：**

    1.  **成功部署：** 應用程式成功部署至 Vercel，並可透過公開 URL 存取。
    2.  **端到端工作流程完成：** 每日工作流程（從 HN 文章擷取到電子報傳送）連續至少 7 天成功且自動完成。
    3.  **手動觸發功能：** 每日工作流程可透過本地 CLI 和遠端安全 API 呼叫成功觸發。
    4.  **功能正常的電子報與使用者介面：** 產生的電子報格式正確且如預期傳送，網頁 UI 的核心功能（顯示電子報、存取 podcast）運作正常。
    5.  **程式碼品質：** 程式碼庫展現良好實踐、結構清晰，並可作為寶貴的學習資源。

## 目標受眾 / 使用者

此應用程式的主要使用者為：

* **工程師：** 具有技術背景，有興趣隨時掌握 Hacker News 最新動態，並學習應用程式架構和 AI 實作的個人。
* **忙碌的科技專業人士：** 科技產業的專業人士，時間有限但希望閱覽當日 Hacker News 熱門 story 的簡明摘要。

這些使用者共同的需求是，希望以簡化的方式隨時了解科技社群內的相關討論和新聞，而無需花費過多時間直接瀏覽 Hacker News。

## 主要功能 / 範疇（MVP 的高階構想）

* **HN 內容擷取與儲存：**

    * 每日使用 HN Algolia API 擷取 Hacker News 排名前 30 的文章及其相關留言。
    * 每日抓取並儲存最多 10 篇連結文章。
    * 儲存所有擷取的資料（文章、留言、報導），並與日期關聯。
* **AI 驅動摘要：**

    * AI 驅動摘要 10 篇選定文章（2 段摘要）。
    * AI 驅動摘要 10 篇選定文章的留言（2 段摘要，突顯有趣的互動）。
    * 透過環境變數設定本地或遠端 LLM 使用。
* **電子報產生與傳送：**

    * 產生每日 HTML 格式電子報，包含摘要、HN 文章和報導連結，以及原始文章日期/時間。
    * 自動將電子報傳送至 Supabase 中手動設定的訂閱者清單。
* **Podcast 產生與整合：**

    * 與 Play.ht's PlayNote API 整合，從電子報內容 AI 生成 podcast。
    * Webhook 處理程式，用以更新電子報並附上產生的 podcast 連結。
* **網頁介面 (MVP)：**

    * 顯示當期與過往電子報。
    * 在網頁內閱讀電子報內容的功能。
    * 電子報下載選項。
    * 用於收聽所產生 podcast 的網頁播放器。
* **API 與觸發：**

    * 安全的 API 端點，用於手動觸發每日工作流程，並以 API 金鑰保護。
    * CLI 指令，用於在本地手動觸發每日工作流程。

## MVP 後續功能 / 範疇與構想

* 使用者驗證與管理：使用者登入、註冊和個人資料管理功能。
* 訂閱管理：允許使用者管理其電子報訂閱的功能（例如修改、暫停或取消）。
* 管理員儀表板：用於管理使用者、電子報和其他應用程式資料的管理介面。
* 增強的電子報自訂功能：供使用者自訂其電子報偏好設定的選項（例如頻率、內容篩選器）。
* 額外的內容摘要：可能根據 Hacker News 類別或其他內容來源提供不同的摘要。
